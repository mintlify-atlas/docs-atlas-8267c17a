## Overview

The `order_created` endpoint creates a new order in the TMT system. This is the first step in the order lifecycle and initializes the order with customer information, tickets, and payment transactions.

<Warning>
  Orders are immediately set to "completed" status upon creation. Ensure all transaction and ticket data is validated before calling this endpoint.
</Warning>

## Order Lifecycle

Orders in the TMT system follow this lifecycle:

1. **Created** - Order is created via `order_created` endpoint with status "completed"
2. **Processing** - Automatically triggered by `order_process` function
3. **Payout** - Payment distribution is calculated and recorded via `order_payout`

## Order States

The order system recognizes the following states:

- **completed** - Order has been successfully created and payment received
- **processing** - Order is being processed (automatic)
- **distributed** - Payment has been distributed to appropriate accounts

<Warning>
  All transactions must have `status: true` and valid payment data. Invalid transactions will cause the order processing to fail.
</Warning>
