## Overview

The `order_process` function is a Firestore trigger that automatically executes when a new order document is created. It handles the complete order processing workflow including ticket updates, transaction generation, and payment distribution.

<Warning>
  This is an automatic background process triggered by order creation. Do not call this function directly.
</Warning>

## Processing Flow

When an order is created, the following steps occur automatically:

1. **Validation** - Verifies order status and transaction amounts
2. **Ticket Update** - Updates ticket status and customer information
3. **Transaction Creation** - Calls `transactions_generate` to create transaction records
4. **User Account Creation** - Creates or updates user account via `order_user`
5. **Payment Distribution** - Calculates and creates payout records based on cost structure

## Payment Distribution Logic

The function implements a sophisticated payment split algorithm:

### Fixed Costs

Fixed costs are collected first from incoming payments:

- If remaining fixed costs >= order amount: 100% goes to fixed costs
- If remaining fixed costs < order amount: Proportional split between fixed and variable

### Variable Costs

After fixed costs are covered, variable costs are distributed:

- **TMT Variable Costs** - Platform fees and operational costs
- **Client Variable Costs** - Event organizer's share

<Warning>
  Payment distribution only occurs for non-TMT point of sale transactions. TMT POS transactions are handled separately through the conciliation process.
</Warning>

## Code Examples

<CodeGroup>

```javascript Understanding the Process
// This function is triggered automatically
// Example of what happens behind the scenes:

const processOrder = async (orderId, orderData) => {
  // 1. Validate order
  if (!orderData.status) {
    console.log('Order not ready for processing');
    return;
  }
  
  // 2. Calculate payment distribution
  const financialSetup = await getEventFinancialSetup(orderData.event_id);
  const distribution = calculateDistribution(
    orderData.amount,
    orderData.transactions,
    financialSetup
  );
  
  // 3. Update tickets with customer info
  for (const ticket of orderData.tickets) {
    await updateTicket(ticket.ticket_id, ticket.metadata);
  }
  
  // 4. Create transaction records
  await createTransactions({
    orderId: orderId,
    transactions: orderData.transactions,
    tickets: orderData.tickets
  });
  
  // 5. Create payout records
  for (const payout of distribution) {
    await createPayoutRecord(payout);
  }
  
  // 6. Update order with distribution
  await updateOrder(orderId, { distribution });
};
```

```javascript Monitoring Order Processing
// You can monitor order processing status
const checkOrderProcessing = async (orderId) => {
  const orderRef = db.collection('orders').doc(orderId);
  const order = await orderRef.get();
  const data = order.data();
  
  if (data.distribution && data.distribution.length > 0) {
    console.log('Order processed successfully');
    console.log('Distribution:', data.distribution);
    
    // Check payout status
    const payouts = await db.collection('orders_payout')
      .where('order_id', '==', orderId)
      .get();
    
    payouts.forEach(payout => {
      const payoutData = payout.data();
      console.log(`${payoutData.entity}: $${payoutData.amount} - Status: ${payoutData.payout_status}`);
    });
  } else {
    console.log('Order still processing...');
  }
};
```

</CodeGroup>

## Financial Setup Requirements

The event must have a financial setup document at:
`events/{event_id}/setup/financial`

Required fields:

<ParamField path="fixed_cost_balance" type="number">
  Total fixed costs for the event
</ParamField>

<ParamField path="fixed_cost_collected" type="number">
  Amount of fixed costs already collected
</ParamField>

<ParamField path="costs_fixed" type="array">
  Array of fixed cost items
  
  <ParamField path="name" type="string">
    Cost name
  </ParamField>
  
  <ParamField path="value_total" type="number">
    Total cost amount
  </ParamField>
</ParamField>

<ParamField path="costs_variables" type="array">
  Array of variable cost percentages
  
  <ParamField path="name" type="string">
    Cost name
  </ParamField>
  
  <ParamField path="entity" type="string">
    "TMT" or "Cliente"
  </ParamField>
  
  <ParamField path="value" type="number">
    Percentage (0-100)
  </ParamField>
</ParamField>

<Warning>
  Variable cost percentages for TMT and Cliente must sum to 100%. Invalid cost structures will cause distribution errors.
</Warning>

## Error Handling

The function silently returns in the following cases:
- Order has no status
- Order snapshot is empty
- Transaction sum doesn't match order amount (commented out in code)

<Warning>
  Failed order processing does not create error records. Monitor order documents for missing `distribution` fields to identify processing failures.
</Warning>
