## Overview

The TMT API provides multiple endpoints for sales metrics and analytics. These endpoints deliver insights into ticket sales, revenue, office performance, and platform statistics.

## Total Sales

### Endpoint
`POST /total_sales`

### Description
Get total sales aggregated by date with overall totals.

### Request Body

<ParamField body="data" type="object" required>
  <ParamField body="event_id" type="string">Filter by event ID</ParamField>
  <ParamField body="from" type="string">Start date (YYYY-MM-DD)</ParamField>
  <ParamField body="to" type="string">End date (YYYY-MM-DD)</ParamField>
</ParamField>

### Response

<ResponseField name="data" type="object">
  <ResponseField name="totales_date" type="array">
    Daily sales breakdown
    <ResponseField name="date" type="string">Sale date</ResponseField>
    <ResponseField name="total" type="number">Total amount for that date</ResponseField>
  </ResponseField>
  <ResponseField name="totales" type="number">Grand total across all dates</ResponseField>
</ResponseField>

### Example

```json
{
  "message": "Total Ventas",
  "status": 200,
  "data": {
    "totales_date": [
      { "date": "2026-03-01", "total": 5420.50 },
      { "date": "2026-03-02", "total": 8350.00 },
      { "date": "2026-03-03", "total": 12100.25 }
    ],
    "totales": 25870.75
  }
}
```

## Tickets vs Tickets Sold

### Endpoint
`POST /number_tickets_vs_sold`

### Description
Compare total available seats with tickets sold and calculate sell-through percentage.

### Request Body

<ParamField body="data" type="object" required>
  <ParamField body="event_id" type="string" required>Event ID</ParamField>
  <ParamField body="from" type="string">Start date</ParamField>
  <ParamField body="to" type="string">End date</ParamField>
</ParamField>

### Response

<ResponseField name="data" type="object">
  <ResponseField name="sold" type="array">
    <ResponseField name="seats_allocated" type="number">Total available seats</ResponseField>
    <ResponseField name="seats_sold" type="number">Tickets sold</ResponseField>
    <ResponseField name="percentage_sold" type="number">Sell-through percentage</ResponseField>
  </ResponseField>
</ResponseField>

### Example

```json

  "status": 200,
  "data": {
    "sold": [
      {
        "seats_allocated": 5000,
        "seats_sold": 3742,

  }

```

## Data Source

All sales metrics query the PostgreSQL database tables:
- `orders` - Order records with ticket arrays
- `orders_transactions` - Payment transactions with currency details
- `orders_payout` - Distribution and payout records

## Common Filters

All endpoints support these optional filters:
- `event_id` - Filter by specific event
- `from` - Start date (YYYY-MM-DD)
- `to` - End date (YYYY-MM-DD)

## Use Cases

- **Revenue tracking**: Monitor daily, weekly, monthly sales
- **Performance analysis**: Compare box offices, events, time periods
- **Capacity planning**: Track sell-through rates
- **Multi-currency reporting**: Separate USD and VEF analytics
- **Client reporting**: Generate client-specific sales reports
