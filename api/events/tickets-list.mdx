## Overview

This endpoint retrieves all tickets associated with a specific event from the PostgreSQL database. It returns complete ticket information including status, customer details, and ledger history.

## Request

<ParamField body="data" type="object" required>
  Request data object
  
  <ParamField body="data.event_id" type="string" required>
    The unique identifier of the event
  </ParamField>
</ParamField>

## Related Endpoints

### Get Individual Ticket

**Endpoint:** `tickets_individual`

```json
{
  "data": {
    "ticket_id": "ABC123XYZ-ticket123"
  }
}
```

### List Tickets for Sales (with blocking status)

**Endpoint:** `tickets_list_event_sales`

Returns tickets with additional blocking information for sales interfaces.

```json
{
  "data": {
    "event_id": "ABC123XYZ"
  }
}
```

This endpoint joins with `tickets_blocked` table to show real-time availability considering temporary locks.

### List Tickets for Access Control

**Endpoint:** `tickets_list_access_control`

Returns only sold tickets (`status = false`) for access control systems.

```json
{
  "data": {
    "event_id": "ABC123XYZ"
  }
}
```

## Code Examples

<CodeGroup>

```bash cURL
curl -X POST https://us-central1-[project-id].cloudfunctions.net/tickets_list_event \
  -H "Content-Type: application/json" \
  -d '{
    "data": {
      "event_id": "ABC123XYZ"
    }
  }'
```

```javascript JavaScript
const listTickets = async (eventId) => {
  const response = await fetch(
    'https://us-central1-[project-id].cloudfunctions.net/tickets_list_event',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        data: {
          event_id: eventId
        }
      })
    }
  );
  
  const result = await response.json();
  return result;
};

// Usage
listTickets('ABC123XYZ')
  .then(data => {
    console.log(`Found ${data.data.response.length} tickets`);
    data.data.response.forEach(ticket => {
      console.log(`${ticket.ticket_id}: ${ticket.status ? 'Available' : 'Sold'}`);
    });
  })
  .catch(error => console.error('Error:', error));
```

```javascript Firebase Functions
const { getFunctions, httpsCallable } = require('firebase/functions');

const functions = getFunctions();
const listTickets = httpsCallable(functions, 'tickets_list_event');

listTickets({ event_id: 'ABC123XYZ' })
  .then((result) => {
    console.log('Tickets:', result.data.data.response);
  })
  .catch((error) => {
    console.error('Error:', error);
  });
```

</CodeGroup>

## Notes

- This endpoint queries the PostgreSQL database for optimal performance with large datasets
- All tickets for the event are returned regardless of status
- For sales interfaces, use `tickets_list_event_sales` which includes blocking information
- For access control systems, use `tickets_list_access_control` which filters to sold tickets only
- The ledger provides a complete audit trail of all ticket actions
