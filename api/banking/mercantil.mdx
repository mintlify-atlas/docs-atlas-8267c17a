## Overview

The Mercantil Banco integration provides support for Venezuelan payment methods including debit cards (TDD), credit cards (TDC), and C2P (Card to Phone) mobile payments. All transactions are processed in Venezuelan Bolivares (VES).

<Warning>
  **Security Notice**: Sensitive data (CVV, OTP codes, phone numbers) are encrypted using AES256 encryption. Never send unencrypted sensitive data.
</Warning>

## Authentication

**Required Headers:**
- `X-IBM-Client-ID`: Your IBM API Gateway client ID
- `Content-Type: application/json`

**Merchant Identification:**
All requests require merchant identification with:
- `integratorId`: 31 (production) or 1 (sandbox)
- `merchantId`: Your assigned merchant ID
- `terminalId`: Terminal identifier

## Payment Methods

<Tabs>
  <Tab title="TDD (Debit Card)">
    Process payments using Venezuelan debit cards with two-factor authentication.

    ### Get Authorization Code

    Request a two-factor authentication code for debit card payments.

    **Endpoint:** `POST /api_mercantil_auth`

    **Parameters:**

    <ParamField body="customer_id" type="string" required>
      Customer ID number (e.g., "V18366876")
    </ParamField>

    <ParamField body="card_number" type="string" required>
      16-digit card number
    </ParamField>

    <ParamField body="ipaddress" type="string" required>
      Customer's IP address
    </ParamField>

    **Request Example:**
    ```json
    {
      "data": {
        "customer_id": "V18366876",
        "card_number": "5018782000662873",
        "ipaddress": "192.168.1.100"
      }
    }
    ```

    **Response:**
    ```json
    {
      "data": {
        "auth_code": "sent",
        "message": "SMS code sent to registered phone"
      },
      "status": 200
    }
    ```

    ---

    ### Process TDD Payment

    Complete a debit card payment with two-factor authentication.

    **Endpoint:** `POST /api_mercantil_tdd_pay`

    **Parameters:**

    <ParamField body="amount" type="number" required>
      Payment amount in VES
    </ParamField>

    <ParamField body="invoice_number" type="string" required>
      Unique invoice identifier
    </ParamField>

    <ParamField body="customer_id" type="string" required>
      Customer ID (e.g., "V18366876")
    </ParamField>

    <ParamField body="card_number" type="string" required>
      16-digit card number
    </ParamField>

    <ParamField body="cvv" type="string" required>
      Card CVV (will be encrypted)
    </ParamField>

    <ParamField body="account_type" type="string" required>
      Account type: "cc" (checking) or "ca" (savings)
    </ParamField>

    <ParamField body="expiration_date" type="string" required>
      Card expiration in format "YYYY/MM"
    </ParamField>

    <ParamField body="twofactor_auth" type="string" required>
      OTP code received via SMS (will be encrypted)
    </ParamField>

    <ParamField body="ipaddress" type="string" required>
      Customer's IP address
    </ParamField>

    **Request Example:**
    ```json
    {
      "data": {
        "amount": 150.50,
        "invoice_number": "INV-98765",
        "customer_id": "V18366876",
        "card_number": "5018782000662873",
        "cvv": "123",
        "account_type": "cc",
        "expiration_date": "2026/12",
        "twofactor_auth": "123456",
        "ipaddress": "192.168.1.100"
      }
    }
    ```

    **Success Response:**
    ```json
    {
      "message": "Procesado",
      "data": {
        "transaction_id": "TRX-123456789",
        "status": "approved",
        "authorization_code": "ABC123",
        "amount": 150.50,
        "currency": "ves"
      }
    }
    ```

## Reconciliation

Automated bank reconciliation for Mercantil transactions.

**Endpoint:** `POST /api_mercantil`

### Process

1. Lists available transaction files from Mercantil TED service
2. Downloads encrypted .7z files containing transaction data
3. Decrypts and parses .cif files
4. Inserts transactions into reconciliation database tables

### Response Format

```json
{
  "message": "Proceso culminado",
  "status": 200,
  "fecha_reg": "20260313",
  "num_cta": "01020105530000767246",
  "saldo_apertura": "150000.50",
  "tipo_detalle": "01",
  "monto_mov": 500.00,
  "tipo_op": "C"
}
```

<Warning>
  **Reconciliation Notes**:
  - Runs automatically for configured date ranges
  - Processes both credit (C) and debit (D) transactions
  - Stores results in `conc_conciliado_b` and `conc_conciliado_c` tables
  - Uses AES256 decryption for file contents
</Warning>

## Error Handling

### Common Errors

| Error | Description | Solution |
|-------|-------------|----------|
| Invalid customer_id | ID format incorrect | Use format: V12345678 or J12345678 |
| Insufficient funds | Account balance too low | Request different payment method |
| Invalid OTP | Two-factor code incorrect | Request new OTP and retry |
| Expired card | Card past expiration date | Request updated card information |
| Blocked card | Card blocked by bank | Contact bank for card status |

### Error Response Format

```json
{
  "message": "Error",
  "error": {
    "code": "INVALID_CARD",
    "message": "La tarjeta ingresada no es válida",
    "details": "Card number failed validation"
  }
}
```

## Integration Example

```javascript
// Example: Process TDD payment with 2FA

// Step 1: Request OTP
const requestAuth = httpsCallable(functions, 'api_mercantil_auth');
const authResult = await requestAuth({
  customer_id: 'V18366876',
  card_number: '5018782000662873',
  ipaddress: '192.168.1.100'
});

// Step 2: Get OTP from user
const otp = prompt('Enter SMS code:');

// Step 3: Process payment
const processPayment = httpsCallable(functions, 'api_mercantil_tdd_pay');
const paymentResult = await processPayment({
  amount: 150.50,
  invoice_number: 'INV-' + Date.now(),
  customer_id: 'V18366876',
  card_number: '5018782000662873',
  cvv: '123',
  account_type: 'cc',
  expiration_date: '2026/12',
  twofactor_auth: otp,
  ipaddress: '192.168.1.100'
});

if (paymentResult.data.data.status === 'approved') {
  console.log('Payment successful!', paymentResult.data.data.transaction_id);
}
```

<Warning>
  **Best Practices**:
  - Always encrypt sensitive data before transmission
  - Validate card numbers and expiration dates before API calls
  - Implement retry logic for network failures
  - Store transaction IDs for reconciliation
  - Handle all error responses gracefully
</Warning>
