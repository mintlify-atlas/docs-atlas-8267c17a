## Overview

This endpoint sends push notifications to mobile app users who have tickets for a specific event. It uses Firebase Cloud Messaging (FCM) to deliver notifications and tracks delivery status in PostgreSQL.

## Notification Flow

1. **Query Users**: Retrieve all users with FCM tokens (`fcmToken != " "`)
2. **Filter by Orders**: Check each user for orders matching the `event_id`
3. **Batch Processing**: Send notifications in batches of 100 users
4. **Track Delivery**: Store notification records in PostgreSQL
5. **Handle Failures**: Update status for failed tokens

## Firebase Cloud Messaging

### Message Structure

```javascript
{
  notification: {
    title: "Notification Title",
    body: "Notification body text",
    image: "https://example.com/image.jpg"
  },
  tokens: ["fcm_token_1", "fcm_token_2", ...] // up to 100 tokens
}
```

### Batch Size
- Maximum 100 tokens per batch
- Automatic batching handled by endpoint
- Prevents FCM quota issues

## Best Practices

1. **Timing**: Send at appropriate times (avoid late night)
2. **Relevance**: Only send to users with tickets
3. **Frequency**: Don't over-notify (causes uninstalls)
4. **Content**: Keep messages concise and actionable
5. **Images**: Use high-quality, relevant images
6. **Testing**: Test on different devices before mass send
7. **Analytics**: Monitor delivery and failure rates

## Error Handling

### Failed Tokens
- Automatically tracked in database
- Status set to `false` for failed deliveries
- Failed tokens logged for investigation
- Common causes:
  - App uninstalled
  - Token expired
  - User disabled notifications
  - Network issues

### Batch Failures
- Individual token failures don't stop batch
- Each batch processes independently
- Failed tokens reported in response

## Performance

- **Batch Size**: 100 users per batch
- **Processing**: Sequential batch processing
- **Database**: Efficient bulk inserts
- **Tracking**: Full delivery audit trail
