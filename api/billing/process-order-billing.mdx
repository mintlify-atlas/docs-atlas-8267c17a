## Overview

This endpoint processes billing for a completed order. It retrieves order details, generates billing data with proper tax calculations (VAT and IGTF), obtains a sequential invoice number, and emits the invoice to the external billing system.

This function is designed to run independently to avoid timeouts in the order processing flow.

## Processing Flow

1. **Retrieve purchaser information** from `u_users` collection if email is provided
2. **Retrieve client (tercero) information** from `u_clients` collection
3. **Obtain sequential invoice number** from `/data/invoice_counter` (atomic transaction)
4. **Generate billing data** with:
   - Ticket items grouped by zone
   - Tax calculations (VAT 16%, IGTF 3% if applicable)
   - Currency conversion (USD to BSD)
   - Amount in words (Spanish)
5. **Emit invoice** to external billing system
6. **Update order** with billing information in Firestore

## Tax Calculation Details

### VAT (IVA) - 16%
- Applied to all ticket sales
- Calculated on subtotal
- Rounded to 2 decimals

### IGTF (Currency Tax) - 3%
- Applied only when payment methods include:
  - Zelle
  - Efectivo Dolares (Cash Dollars)
  - Efectivo Divisa (Foreign Cash)
- Base: Total amount paid in foreign currency
- Capped at invoice total (Subtotal + IVA)
- Truncated to 2 decimals (not rounded)

### Currency Conversion
- Exchange rate from `/data/exchange_rates`
- Individual components truncated to 2 decimals
- Total = Sum of truncated components

## Invoice Number Generation

Invoice numbers are:
- Sequential and atomic (using Firestore transaction)
- Stored in `/data/invoice_counter`
- 8-digit zero-padded format (e.g., "00000123")
- Automatically incremented for each invoice

## Error Codes

- **200**: Success
- **400**: Billing failed (validation errors, external API rejection)
- **500**: Critical error (authentication failure, missing data, system error)
