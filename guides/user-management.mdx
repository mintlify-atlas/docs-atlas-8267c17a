The TMT API provides comprehensive user management capabilities for creating clients, collaborators, staff members, and end users. This guide covers user creation, validation, and management operations.

## User Types

The TMT system manages different types of users:

- **Clients** (`u_clients`): Organizations that host events
- **Collaborators** (`u_collaborators`): Staff members working for clients
- **App Users** (`u_users`): End users who purchase tickets
- **Staff**: Administrative and support personnel

## Creating Clients

Clients are organizations that create and manage events. Use the `create_client` endpoint to register a new client.

<Steps>
  <Step title="Prepare Client Data">
    Gather all required information including business details, contact information, and banking details.
  </Step>
  
  <Step title="Submit Client Creation Request">
    Send a POST request to create the client account.
  </Step>
  
  <Step title="Verify Account Creation">
    Check the response for the generated UID and confirm account status.
  </Step>
</Steps>

## Creating Collaborators

Collaborators are staff members who work for clients and manage events, access control, and other operations.

### Request Example

<CodeGroup>
```javascript Node.js
const collaboratorData = {
  data: {
    name: "Carlos Rodriguez",
    email: "carlos@acmeentertainment.com",
    password: "SecurePass456!",

    id_type: "V",

    country: "Venezuela",
    line: "Calle 5, Edificio B",
    zip_code: "1060"
  }
};

const response = await axios.post(
  'https://your-api-url/create_collaborator',
  collaboratorData

```

<Warning>

</Warning>

## App User Management

App users are end customers who purchase tickets through the platform. They can be created automatically during the order process or manually.

### Validate App User

Validate a user's security code:

```javascript
const validateUser = {
  data: {
    email: "customer@example.com",
    security_cod: "123456"
  }
};

const response = await axios.post(
  'https://your-api-url/app_user_validate',
  validateUser
);
```

### Response

```json
{
  "message": "Codigo Valido",
  "status": 200,
  "data": {
    "valido": true
  }
}
```

## Ticket Transfer

Transfer tickets between users:

<Steps>
  <Step title="Validate Recipient">
    Ensure the recipient email is registered and validated.
  </Step>
  
  <Step title="Initiate Transfer">
    Submit the transfer request with ticket ID and recipient email.
  </Step>
  
  <Step title="Update Ledger">
    The system automatically updates the ticket ledger with transfer information.
  </Step>
</Steps>

### Online Transfer

```javascript
const transferRequest = {
  data: {
    email: "recipient@example.com",
    ticket_id: "event789-ticket001",
    email_from: "sender@example.com"
  }
};

const response = await axios.post(
  'https://your-api-url/app_transfer_tickets',
  transferRequest
);
```

### Offline Transfer

For transfers made offline and synced later:

```javascript
const offlineTransfer = {
  data: {
    email: "recipient@example.com",
    ticket_id: "event789-ticket001",
    email_from: "sender@example.com"
  }
};

const response = await axios.post(
  'https://your-api-url/app_transfer_tickets_offline',
  offlineTransfer
);
```

<Warning>
Ticket transfers require:
- Recipient must have a validated account
- Ticket must not have been previously transferred
- Original owner must initiate the transfer
</Warning>

## Best Practices

<CardGroup cols={2}>
  <Card title="Secure Passwords" icon="lock">
    Always use strong passwords with minimum 8 characters including uppercase, lowercase, numbers, and special characters.
  </Card>
  
  <Card title="Validate Users" icon="check">
    Implement email validation to ensure users can receive important notifications and ticket information.
  </Card>
  
  <Card title="Store UIDs Safely" icon="database">
    Keep user UIDs secure and never expose them in client-side code or public URLs.
  </Card>
  
  <Card title="Handle Errors Gracefully" icon="triangle-exclamation">
    Always check response status codes and handle errors appropriately in your application.
  </Card>
</CardGroup>
