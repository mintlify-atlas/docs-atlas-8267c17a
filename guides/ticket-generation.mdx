Ticket generation is a core feature of the TMT platform. This guide covers creating tickets, managing inventory, implementing temporary locks during checkout, and tracking ticket lifecycle.

## Ticket Generation Process

Tickets are generated based on the zone configuration you created during event setup. Each ticket is assigned a unique ID and tracked through its entire lifecycle.

<Steps>
  <Step title="Configure Event Zones">
    Ensure your event has zones configured with seat counts and pricing.
  </Step>
  
  <Step title="Generate Tickets">
    Call the ticket generation endpoint to create all tickets for the event.
  </Step>
  
  <Step title="Verify Creation">
    Confirm all tickets were created successfully in both Firestore and PostgreSQL.
  </Step>
  
  <Step title="Activate Sales">
    Make tickets available for purchase by updating event status.
  </Step>
</Steps>

## Generate Tickets for Event

Generate all tickets for an event based on zone configuration:

```javascript
const response = await axios.post(
  'https://your-api-url/tickets_generate',
  {
    data: {
      idevent: "evt_2024_concert_001"
    }
  }
);
```

### How It Works

The ticket generation process:

1. Reads event details from Firestore
2. Retrieves zone configuration from `events/{eventId}/setup/zones`
3. Iterates through each zone and creates tickets based on seat count
4. Assigns unique IDs to each ticket
5. Writes tickets to both Firestore and PostgreSQL
6. Creates initial ledger entry with "generated" action

### Response

```json
{
  "message": "Se Generaron: 5000",
  "status": 200,
  "data": {
    "valido": true
  }
}
```

<Tip>
Ticket generation is a one-time operation per event. Once tickets are generated, they cannot be regenerated. Plan your zones carefully before generating tickets.
</Tip>

## Ticket Locking (Reservation)

Temporarily lock tickets during the checkout process to prevent double-booking:

```javascript
const lockTickets = {
  data: {
    ticketd_reserved: [
      "evt_2024_concert_001-abc123xyz",
      "evt_2024_concert_001-def456uvw"
    ],
    reserved_time: 900  // 15 minutes in seconds
  }
};

const response = await axios.post(
  'https://your-api-url/tickets_lock',
  lockTickets
);
```

### How Locking Works

1. System checks if all tickets are available
2. Checks if tickets are already locked by another user
3. If available, creates lock records in `tickets_blocked` table
4. Sets expiration time based on `reserved_time` parameter
5. Tickets remain locked until purchased or lock expires

### Success Response

```json
{
  "message": "Tickets blocked",
  "status": 200,
  "data": {
    "valido": true,
    "tickets": [
      "evt_2024_concert_001-abc123xyz",
      "evt_2024_concert_001-def456uvw"
    ]
  }
}
```

### Error Response (Tickets Not Available)

```json
{
  "message": "Tickets no disponibles",
  "status": 400,
  "data": {
    "valido": false,
    "nodisponible": [
      "evt_2024_concert_001-abc123xyz"
    ]
  }
}
```

<Warning>
Always implement ticket locking during checkout to prevent race conditions where multiple users try to purchase the same ticket simultaneously.
</Warning>

## Ticket Unlocking

Unlock tickets when users abandon cart or locks expire:

<CodeGroup>
```javascript Automatic Unlock (Expired Locks)
const response = await axios.post(
  'https://your-api-url/tickets_unlock',
  {
    data: {}
  }
);
// Automatically unlocks all tickets with expired locks
```

```javascript Manual Unlock (Specific Tickets)
const response = await axios.post(
  'https://your-api-url/tickets_unlock_param',
  {
    data: {
      ticketd_reserved: [
        "evt_2024_concert_001-abc123xyz",
        "evt_2024_concert_001-def456uvw"
      ]
    }
  }
);
```
</CodeGroup>

### When to Unlock Tickets

- User closes checkout page
- User explicitly cancels reservation
- Payment fails
- Lock timer expires (automatic)
- User session times out

## Ticket Ledger

Every ticket maintains a ledger that tracks its complete history:

### Ledger Actions

- `generated` - Ticket was created
- `reserved` - Ticket was temporarily locked
- `sold` - Ticket was purchased
- `assigned` - Ticket was assigned to customer
- `transferred` - Ticket ownership was transferred
- `accessed` - Ticket was scanned at entry
- `came-out` - Ticket holder exited venue
- `updated` - Ticket information was modified

### Example Ledger

```json
{
  "ledger": [
    {
      "date": "2024-03-10T10:00:00Z",
      "action": "generated",
      "metadata": ""
    },
    {
      "date": "2024-03-11T14:25:00Z",
      "action": "reserved",
      "metadata": "{\"reserved_by\":\"user123\",\"expires\":\"2024-03-11T14:40:00Z\"}"
    },
    {
      "date": "2024-03-11T14:30:00Z",
      "action": "sold",
      "metadata": "{\"order_id\":\"order456\",\"payment_method\":\"stripe\"}"
    },
    {
      "date": "2024-07-15T18:30:00Z",
      "action": "accessed",
      "metadata": "{\"checkpoint\":\"main_entrance\",\"staff_id\":\"staff789\"}"
    }
  ]
}
```

<Tip>
The ledger is immutable - new actions are appended, never modified. This provides a complete audit trail for each ticket.
</Tip>

## Inventory Management

### Check Available Tickets by Zone

```javascript
const getAvailableByZone = async (eventId) => {
  const tickets = await db.collection('events')
    .doc(eventId)
    .collection('tickets')
    .where('status', '==', true)
    .get();
  
  const inventory = {};
  
  tickets.forEach(ticket => {
    const data = ticket.data();
    if (!inventory[data.zone]) {
      inventory[data.zone] = 0;
    }
    inventory[data.zone]++;
  });
  
  return inventory;
};

// Result:
// {
//   "VIP": 450,
//   "General Admission": 3200,
//   "Balcony": 980
// }
```

### Check Total Sold Tickets

```javascript
const getSoldCount = async (eventId) => {
  const soldTickets = await db.collection('events')
    .doc(eventId)
    .collection('tickets')
    .where('status', '==', false)
    .get();
  
  return soldTickets.size;
};
```

## Common Scenarios

### Scenario 1: User Checkout Flow

```javascript
// 1. User selects tickets
const selectedTickets = [
  "evt_2024_concert_001-abc123xyz",
  "evt_2024_concert_001-def456uvw"
];

// 2. Lock tickets (15 minutes)
const lockResponse = await axios.post(
  'https://your-api-url/tickets_lock',
  {
    data: {
      ticketd_reserved: selectedTickets,
      reserved_time: 900
    }
  }
);

if (lockResponse.data.status === 200) {
  // 3. Process payment
  // 4. On success, tickets are marked as sold during order creation
  // 5. Lock is automatically removed
} else {
  // Tickets not available, show error
  alert('Some tickets are no longer available');
}
```

### Scenario 2: Abandoned Cart Cleanup

```javascript
// Run periodically (every 5 minutes)
setInterval(async () => {
  await axios.post(
    'https://your-api-url/tickets_unlock',
    { data: {} }
  );
  console.log('Expired locks cleared');
}, 300000);
```

### Scenario 3: Check Availability Before Purchase

```javascript
const checkAvailability = async (ticketIds) => {
  const response = await axios.post(
    'https://your-api-url/tickets_lock',
    {
      data: {
        ticketd_reserved: ticketIds,
        reserved_time: 60  // 1 minute test lock
      }
    }
  );
  
  if (response.data.status === 200) {
    // Tickets available, unlock them
    await axios.post(
      'https://your-api-url/tickets_unlock_param',
      {
        data: { ticketd_reserved: ticketIds }
      }
    );
    return true;
  }
  
  return false;
};
```
