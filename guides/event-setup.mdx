Event setup is the foundation of the TMT ticketing system. This guide walks you through creating events, configuring zones with different pricing tiers, and setting up seating arrangements.

## Event Lifecycle

Events in TMT go through several status states:

1. **Instantiated** - Event is created
2. **Active** - Event is open for ticket sales
3. **In Progress** - Event is currently happening
4. **Finished** - Event has concluded
5. **Cancelled** - Event was cancelled

## Creating an Event

Before generating tickets, you need to create an event with all necessary details.

## Financial Configuration

Set up the cost structure for your event to enable proper revenue splitting.

### Fixed Costs

Fixed costs are one-time expenses that must be covered before variable costs:

```json
{
  "fixed_cost_balance": 10000.00,
  "fixed_cost_collected": 0.00,
  "costs_fixed": [
    {
      "name": "Venue Rental",
      "value_total": 5000.00,
      "entity": "TMT"
    },
    {
      "name": "Equipment Rental",
      "value_total": 3000.00,
      "entity": "TMT"
    },
    {
      "name": "Insurance",
      "value_total": 2000.00,
      "entity": "TMT"
    }
  ]
}
```

### Variable Costs

Variable costs are percentage-based and split between TMT and the client:

```json
{
  "costs_variables": [
    {
      "name": "Platform Fee",
      "value": 10,
      "entity": "TMT",
      "description": "TMT service fee"
    },
    {
      "name": "Payment Processing",
      "value": 3,
      "entity": "TMT",
      "description": "Credit card and payment gateway fees"
    },
    {
      "name": "Revenue Share",
      "value": 87,
      "entity": "Cliente",
      "description": "Client's share of revenue"
    }
  ]
}
```

## Automatic Status Updates

The system can automatically update event status based on dates:

```javascript
const response = await axios.post(
  'https://your-api-url/update_events_status',
  {}
);
```

This endpoint checks all events and:
- Changes events with `date_end` in the past from "Activo" to "Finalizado"
- Runs periodically to keep event statuses current

## Complete Event Setup Example

Here's a complete workflow to set up an event:

```javascript
const setupEvent = async (eventData) => {
  const eventId = 'evt_2024_concert_001';
  
  // 1. Create main event document
  await db.collection('events').doc(eventId).set({
    event: {
      name: eventData.name,
      description: eventData.description
    },
    client_id: eventData.clientId,
    client_name: eventData.clientName,
    date_start: eventData.dateStart,
    date_end: eventData.dateEnd,
    status: 'Instantiated',
    type: eventData.type,
    date: {
      created: admin.firestore.Timestamp.now(),
      updated: ''
    }
  });
  
  // 2. Configure zones
  await db.collection('events').doc(eventId)
    .collection('setup').doc('zones').set({
      status: true,
      seats_allocated: eventData.totalSeats,
      zone: eventData.zones
    });
  
  // 3. Configure financial structure
  await db.collection('events').doc(eventId)
    .collection('setup').doc('financial').set({
      fixed_cost_balance: eventData.fixedCosts,
      fixed_cost_collected: 0,
      costs_fixed: eventData.fixedCostItems,
      costs_variables: eventData.variableCosts
    });
  
  // 4. Set up security
  await db.collection('events').doc(eventId)
    .collection('security').doc('qr-pin').set({
      data: {
        qr_pin: generatePin(),
        access_pin: generateAccessPin()
      }
    });
  
  // 5. Activate event
  await db.collection('events').doc(eventId).update({
    status: 'Activo',
    'date.updated': admin.firestore.Timestamp.now()
  });
  
  return eventId;
};
```
